# Recruiting Guide

> 1·2학년 중심으로 장기 성장 가능한 팀원을 발굴하기 위한 모집 문서입니다.  
> 목표는 이름만 올리는 팀원이 아니라, 실제 역할을 맡고 포트폴리오를 남길 수 있는 팀원을 찾는 것입니다.

---

## 우리가 찾는 사람

완성된 실력자보다 오래 같이 성장할 사람을 우선합니다.

좋은 팀원 기준:

- 약속을 지키는 사람
- 모르는 것을 부끄러워하지 않고 배우는 사람
- 작은 일이라도 끝까지 해보는 사람
- GitHub, 문서화, 발표자료에 흔적을 남길 수 있는 사람
- 자기 전공을 프로젝트에 연결하려는 사람

## 1·2학년 온보딩 방식

### 1단계

README 읽기  
프로젝트 목적 이해  
GitHub 계정 준비  

### 2단계

자기 전공과 연결되는 역할 선택  
관련 기술 1페이지 요약  

### 3단계

작은 작업 하나 맡기  
예: 문서 정리, 화면 캡처, API 실행, 데이터 조사  

### 4단계

작업 로그 작성  
회의록 또는 실험 로그 남기기  

### 5단계

GitHub Issue / PR / 발표자료에 이름 남기기

## 팀 운영 원칙

이름만 올리는 팀원은 받지 않는다.  
잘 몰라도 배우려는 사람은 받는다.  
말보다 기록과 결과물을 우선한다.  
작게라도 맡은 일을 끝내는 사람을 우선한다.
