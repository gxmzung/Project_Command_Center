# Role & Skill Tracks

팀원 역할은 이름만 적는 것이 아니라, 무엇을 공부시키고 어떤 산출물을 맡길지까지 연결해야 한다.

## 1. Project Lead / PM

### 하는 일
- 전체 일정 관리
- 회의 진행
- GitHub Issues 정리
- 발표 흐름 설계
- 팀원 작업물 통합

### 필요한 공부
- GitHub Projects
- 요구사항 정의
- 일정 관리
- 발표/제안서 구조
- 리스크 관리

### 산출물
- Master Schedule
- 회의록
- 역할 분담표
- 최종 발표 흐름
- 제출 체크리스트

## 2. Backend / API

### 하는 일
- 서버 API 설계
- DB 모델 설계
- 관리자 기능 구현
- 로그/에러 처리
- 인증/권한 구조 구현

### 필요한 공부
- FastAPI
- REST API
- SQLite/PostgreSQL
- 인증 구조
- API 문서화

### 산출물
- API 명세서
- 서버 코드
- DB 스키마
- 테스트 로그
- 실행 Runbook

## 3. Frontend / Web

### 하는 일
- 관리자 화면
- 대시보드 화면
- 사용자 화면
- 데이터 시각화
- 반응형 UI

### 필요한 공부
- HTML/CSS/JS
- React
- Chart.js/Recharts
- UI/UX 기본
- 상태 관리

### 산출물
- 화면 캡처
- 페이지 코드
- UI 플로우
- 접근성 개선 기록

## 4. Mobile / App

### 하는 일
- Android 앱 화면 구현
- API 연동
- 사용자 플로우 구현
- 로컬 설정 관리
- 빌드/테스트

### 필요한 공부
- Kotlin
- Jetpack Compose
- Retrofit/OkHttp
- Android 권한
- Gradle 빌드 구조

### 산출물
- APK 또는 실행 캡처
- 주요 화면
- API 연동 코드
- 빌드 로그

## 5. Data / Analytics

### 하는 일
- 설문 데이터 정리
- 운영 데이터 분석
- 지표 설계
- 예측/분류 모델 실험
- 대시보드 지표 정의

### 필요한 공부
- Python
- Pandas
- 데이터 전처리
- 기초 통계
- 시각화

### 산출물
- 분석 노트북
- CSV 정리본
- 그래프
- 인사이트 요약

## 6. DevOps / GitHub Ops

### 하는 일
- GitHub Projects 관리
- Issue/PR 규칙 관리
- Actions 자동화
- 배포/실행 문서화
- 증거자료 구조화

### 필요한 공부
- Git
- GitHub CLI
- GitHub Actions
- Markdown 문서화
- Docker 기초

### 산출물
- Workflow 파일
- Issue Template
- Project Field Schema
- Runbook
- Evidence Package

## 7. Hardware / Embedded

### 하는 일
- 센서/보드/장비 구성
- 통신 테스트
- 펌웨어 또는 제어 코드
- 실험 로그 기록
- 하드웨어-소프트웨어 인터페이스 정리

### 필요한 공부
- Arduino/STM32
- UART/I2C/SPI
- 센서 데이터 처리
- 회로 기본
- 테스트 로그 작성

### 산출물
- 회로/배선도
- 실험 사진
- 펌웨어 코드
- 테스트 결과표
- HW/SW 인터페이스 문서

## 8. Research / Documentation

### 하는 일
- 논문/자료 조사
- 참고문헌 정리
- 발표 대본 작성
- 제안서 문장 다듬기
- 근거 자료 축적

### 필요한 공부
- 자료조사법
- 논문 읽기
- 인용 정리
- 보고서 구조
- 발표 스토리라인

### 산출물
- 참고문헌 목록
- 조사 요약
- 발표 대본
- 보고서 초안
- Q&A 예상질문
